# 01. 世界观与节点结构 (World Structure)

## 1. 业务模型：无限层级的创作树

Storyteller 的核心业务实体不再是单一的“文档”，而是一棵**可生长的知识树**。世界观是一个巨大的组合体，由无数个原子化的子内容组成。

### 1.1 结构层级示例
一个典型的奇幻世界观结构如下：

- **🌍 世界观根节点 (World Root)**
  - **🏔️ 地理与环境 (Category)**
    - 📄 地形设定档 (KV: 大洲、海洋)
    - 🗺️ 世界地图 (Image)
    - 🌪️ 气候与灾害 (KV)
  - **🏰 社会与文明 (Category)**
    - ⚖️ 政治制度 (KV)
    - 💰 经济贸易 (KV)
    - ⚔️ 势力关系图 (Graph/Renderable)
  - **🐉 生物与生态 (Category)**
    - 🌿 植被大全 (KV)
    - 👹 怪物图鉴 (Folder)
      - 🐺 恐狼 (Character Card)
      - 🐲 巨龙 (Character Card)
  - **📜 历史与传承 (Category)**
    - ⏳ 编年史 (Timeline)
    - 🦸 英雄传说 (Text)

## 2. 节点类型定义

### 2.1 组合节点 (Composite Node)
- **定义**：作为容器存在，用于组织和分类子内容。
- **业务表现**：
  - UI 上表现为文件夹或分类标题。
  - 支持“展开/折叠”。
  - **聚合能力**：可以自动汇总子节点的内容，生成“分类摘要”（例如：自动生成“地理总述”）。

### 2.2 原子节点 (Leaf Node)
- **定义**：承载具体创作内容的最小单元。
- **业务表现**：
  - UI 上表现为具体的编辑器（KV 表单、文本框、图片预览）。
  - 拥有独立的版本历史 (Version History)。
  - 是 AI 生成和修改的主要对象。

## 3. 业务扩展性

这种树状结构允许业务无限扩展：
- **新增分类**：用户可以随时右键新增一个“神秘组织”分类。
- **混合排版**：在“历史”分类下，可以同时存在“时间轴节点”和“战役插图节点”。
- **模板化**：系统提供预设的“世界观骨架”（Seed），用户开局即获得一套标准结构，但随后可自由修改。
