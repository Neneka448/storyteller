# 06. 创作体验与 UI 指南（Authoring UX Guidelines）

目标：让用户感觉在“写条目/写设定”，而不是“填后台表”。

## 1. 体验原则（必须坚持）

- **阅读流优先**：默认像一页可阅读的百科/卡片；编辑是“点击即改”，不打断阅读。
- **少线框、少按钮**：避免大量输入框边框与显眼按钮。
- **空态即引导**：内容为空时，不展示一堆空控件，展示“可写什么/从哪里开始”。
- **Prompt 是产品**：每个关键字段/文本块都应该有高质量引导语（也可被 AI 使用）。
- **用户永远可自由写**：模板再结构化，也要保留自由文本块与自定义字段（但不要把它们当成默认）。

## 2. 页面结构建议（百科感/维基感）

### 2.1 分类页（Composite Node）

分类页应该回答：
- 这个分类是什么（概述）
- 下面有哪些条目（子节点列表）
- 下一步写什么（建议/待办）

推荐 Section 组合：
- Header：标题 + 可选头图
- ChildrenList：按类型/顺序列出子节点（支持快速创建）
- MemoBlock：分类概述（短文）
- KvGroup（可选）：分类元信息（例如“地区范围/时代/关键主题”）

### 2.2 条目页（Leaf Node）

条目页应该同时满足“可读”与“可检索/可结构化”：
- 左侧（Main）：MemoBlock（概述/历史/文化…）
- 右侧（Sidebar）：KvGroup（信息卡：关键词、数值、标签、引用）
- 图片：头图或插图作为视觉锚点

## 3. KvGroup 的 UI 不是 KV 表

- 显示为“信息卡/字段列表”，每行是 `label : value`，点击 value 进入编辑态
- 支持字段分组（基础/政治/经济/军事…）
- 支持 reference 字段（未来）：自动补全并跳转
- 支持“其他/自定义”折叠组（展示模板外字段，避免数据丢失）

## 4. MemoBlock 的 UI

- 默认是“文章视图”：段落排版良好
- 点按进入编辑：尽量轻工具栏/可选 markdown
- 空态显示 prompt：例如“用 3 句话描述这里的气候与灾害”

## 5. AI 协作的体验约束（只列 UX 层）

- 默认按钮语义：**生成草稿**（不会覆盖现有内容）
- 显式操作：采纳（Adopt）
- 需要对比：至少在版本列表里可回退/可选择

## 6. MVP 范围（避免过度设计）

优先做：
- 分类页 ChildrenList + 概述 MemoBlock
- 条目页 Sidebar KvGroup + Main MemoBlock
- prompt 与空态

后续再做：
- 关系图/时间轴/地图的专用编辑器
- 高级字段类型（单位、枚举、引用约束、校验规则）
